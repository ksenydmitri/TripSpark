# Отчет по Лабораторной работе №6

---

## 1. Введение и цели работы
Целью данной лабораторной работы является качественный аудит и последующий рефакторинг пользовательского интерфейса (UI) и пользовательского опыта (UX) приложения **TripSpark**. Работа опирается на международные стандарты (ISO/IEC 25010:2011), спецификации доступности (WCAG 2.0) и принципы многоуровневого проектирования опыта использования.

В данном этапе разработки фокус был сделан на переходе от линейной навигации к циклической (через Bottom Navigation), улучшении информационной архитектуры и обработке граничных состояний (Empty States).

---

## 2. Оценка разрабатываемого ПО по атрибутам качества «Usability» (ISO/IEC 25010:2011)

До проведения работ по улучшению, приложение было проанализировано по ключевым атрибутам качества группы Usability (SQuaRE):

1.  **Распознаваемость соответствия (Appropriateness recognizability) — 5/5**
    *   *Оценка:* Использование стандартных иконок Material Design (карта, сердце для избранного, профиль) позволяет пользователю мгновенно соотнести элементы интерфейса с их функциями.
2.  **Обучаемость (Learnability) — 3/5 (ДО) ➔ 5/5 (ПОСЛЕ)**
    *   *Проблема:* При окончании списка рекомендаций экран оставался пустым, что могло быть воспринято как ошибка загрузки.
    *   *Решение:* Внедрение состояния «Empty State» в `RecommendationScreen` с текстом «Больше нет рекомендаций», что дает пользователю четкую обратную связь.
3.  **Используемость / Операбельность (Operability) — 2/5 (ДО) ➔ 5/5 (ПОСЛЕ)**
    *   *Проблема:* Навигация между разделами (Интересы, Карта, Вишлист) была затруднена и требовала последовательного возврата по стеку переходов.
    *   *Решение:* Реализация глобальной навигационной панели `NavigationBar` (Bottom Bar). Теперь основные функции доступны в один тап из любой точки приложения.
4.  **Защита от ошибок пользователя (User error protection) — 4/5 (ДО) ➔ 5/5 (ПОСЛЕ)**
    *   *Улучшение:* Использование `launchSingleTop = true` и `restoreState = true` в логике навигации предотвращает наслоение одинаковых экранов в стеке при многократном нажатии на одну и ту же иконку меню.
5.  **Эстетика GUI (User interface aesthetics) — 3/5 (ДО) ➔ 5/5 (ПОСЛЕ)**
    *   *Проблема:* Контент накладывался на системные элементы (Status Bar), отсутствовала визуальная иерархия кнопок.
    *   *Решение:* Добавлены отступы (`padding(top = 32.dp)`), внедрен `Scaffold` для правильной компоновки, кнопки в рекомендациях теперь сгруппированы в аккуратный `Row` с использованием `MaterialTheme.colorScheme`.
6.  **Доступность (Accessibility / WCAG 2.0) — 2/5 (ДО) ➔ 5/5 (ПОСЛЕ)**
    *   *Проблема:* Иконки в навигации не имели текстовых описаний для скринридеров.
    *   *Решение:* Каждому элементу `NavigationBarItem` и новым кнопкам добавлены корректные `contentDescription` и текстовые метки (label).

## 3. Декомпозиция по уровням «Опыта использования» (Jesse James Garrett's 5 Elements of UX)

### Уровень 1: Стратегии (Strategy)
*   **Потребности пользователей:** Быстрый поиск идей для путешествий, сохранение интересных мест и визуализация их на карте.
*   **Бизнес-цели:** Удержание пользователя за счет удобства планирования маршрутов (переход от карточки к карте в один клик).

### Уровень 2: Набора возможностей (Scope)
*   **Функционал:** Подбор рекомендаций на основе интересов, «умная» карта с маркерами релевантности, управление списком желаний (Wishlist), профиль пользователя.

### Уровень 3: Структуры (Structure)
*   **Информационная архитектура:** Переход к плоской структуре навигации. `MainActivity` теперь выступает в роли координатора, управляя `NavHost` внутри `Scaffold`. Основные узлы: Auth ➔ [Interests | Ideas | Map | Wishlist | Profile].

### Уровень 4: Компоновки (Skeleton)
*   **Расположение элементов:**
    *   *Bottom Bar:* Постоянный доступ к 5 ключевым разделам.
    *   *Action Area:* В `RecommendationScreen` кнопки «Вишлист» и «На карту» расположены симметрично в нижней части экрана, что удобно для управления одной рукой.

### Уровень 5: Поверхности (Surface)
*   **Визуальный дизайн:** Цветовая палитра адаптирована под `Material3`. Использование `MaterialTheme.colorScheme.outline` для границ аватара и `primary` для индикаторов загрузки обеспечивает визуальную целостность системы.

## 4. Обоснование и реализация UX-улучшений («ДО» vs «ПОСЛЕ»)

### Улучшение №1: Глобальная навигация (Bottom Navigation)
*   **Необходимость:** Пользователям нужно часто переключаться между списком рекомендаций и картой.
*   **Реализация:** В `MainActivity.kt` добавлен компонент `NavigationBar`.
*   **Результат:** Interaction Cost снижен до минимума. Добавлены русскоязычные подписи: «Интересы», «Идеи», «Карта», «Желания», «Профиль».

### Улучшение №2: Информативность карты (Marker Refinement)
*   **Необходимость:** Маркеры на карте были неинтуитивны.
*   **Реализация:** В `MapScreen.kt` добавлены поясняющие сниппеты на русском языке («В списке желаемого», «Релевантность: X%»). Скорректирована прозрачность (`alpha`) маркеров для визуального выделения наиболее подходящих мест.
*   **Результат:** Карта стала читаемой без дополнительных инструкций.

### Улучшение №3: Обработка пустого состояния (Empty States)
*   **Необходимость:** Избежать эффекта «сломанного приложения», когда рекомендации заканчиваются.
*   **Реализация:** Условный рендеринг в `RecommendationScreen.kt` (проверка `cards.isEmpty()`).
*   **Результат:** Пользователь получает вежливое уведомление «Больше нет рекомендаций» вместо пустого экрана.
